## Архитектура модели

### Внутренний блок — `ReboilerProcess`
Система ОДУ из 3 уравнений (баланс массы, компонента, энергии):

```
dM/dt  = F_in - F_out - F_vap          # без самовыравнивания!
d(Mx)/dt = F_in·x_in - F_out·x - F_vap·y
dU/dt  = Q_steam - Q_loss - F_vap·H_vap
```

**Входы управления (MV):**
| Переменная | Смысл |
|---|---|
| `v_in` | Клапан питания [0..1] |
| `v_out` | Клапан выхода жидкости [0..1] |
| `F_steam` | Расход греющего пара [0..1] |

**Возмущения (DV):**
| Переменная | Смысл |
|---|---|
| `x_in` | Мольная доля лёгкого компонента в питании |
| `T_in` | Температура питания, К |

**Выходы (PV):**
| Переменная | Смысл |
|---|---|
| `L` | Уровень жидкости, м |
| `T` | Температура, К |
| `x` | Состав жидкости |

### Внешний блок — `PIDController` + `Simulator`
- ПИД по уровню → управляет `v_out`
- ПИД по температуре → управляет `F_steam`
- `v_in` задаётся как номинал

### Неисправности — `FaultInjector`
| ID | Название | Тип |
|---|---|---|
| 0 | Норма | — |
| 1 | Скачок состава питания | Ступенчатое возмущение |
| 2 | Медленный дрейф состава | Линейный дрейф |
| 3 | Засорение клапана питания | Деградация клапана |
| 4 | Утечка теплоносителя | Потеря усиления |
| 5 | Залипание клапана выхода | Клапан застрял |
| 6 | Шум датчика уровня | Сенсорный шум |
| 7 | Смещение датчика температуры | Сенсорное смещение |

## Бенчмарк

Три метода детектирования:
1. **PCA** — T²-статистика Хотеллинга + Q (SPE), порог 99%
2. **CUSUM** — накопительная сумма, чувствителен к малым дрейфам
3. **Z-score** — простейший пороговый детектор

Метрики для каждой неисправности:
- **FAR** — False Alarm Rate (ложные тревоги на нормальном режиме)
- **DR** — Detection Rate (доля обнаружений после начала неисправности)
- **ADD** — Average Detection Delay (задержка первого обнаружения, с)
